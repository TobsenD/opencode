# Threat Models

This directory contains security threat models and vulnerability assessments for the OpenCode project.

## Structure

Threat models are organized by:
- **Date**: YYYY-MM-DD format for when the assessment was conducted
- **Type**: Either `pytm` (OWASP pytm) or `threagile` format
- **Scope**: Project area or component being assessed

## Examples

- `2024-03-31-pytm-container-security.md` - Threat model for container security
- `2024-03-31-pytm-supply-chain.md` - Supply chain vulnerability assessment
- `2024-03-31-pytm-git-handling.md` - Git credentials and config handling

## Generation

Threat models are generated by the `security_engineer` agent using:
1. **OWASP pytm** - For data flow threat modeling
2. **Threagile** - For application architecture threat analysis

All models include:
- Threat description and impact assessment
- CVSS scoring when applicable
- Remediation recommendations
- References to security best practices

## Usage

Run the security engineer agent to generate new threat models:
```bash
opencode security_engineer "Analyze threat model for [component]"
```

Findings will be automatically stored in this directory with timestamps.
